## Git 版本控制

### 常用命令
<Tabs>
  <TabItem label="基础操作">
  ```bash
  # 添加修改
  git add .
  
  # 提交修改
  git commit -m "更新说明"
  
  # 推送到远程
  git push
  ```
  </TabItem>
  <TabItem label="分支操作">
  ```bash
  # 创建并切换到新分支
  git checkout -b test-feature
  
  # 切换回主分支
  git checkout main
  
  # 合并分支
  git merge test-feature
  ```
  </TabItem>
  <TabItem label="版本回滚">
  ```bash
  # 回滚到上一次提交
  git reset --hard HEAD^
  
  # 回滚到指定版本
  git reset --hard 版本号
  
  # 给重要的版本打标签
  git tag v1.0 -m "第一个可用版本"
  ```
  </TabItem>
  <TabItem label="标签管理">
  ```bash
  # 创建标签
  git tag v1.0 -m "第一个可用版本"
  
  # 查看所有标签
  git tag
  
  # 推送标签到远程
  git push origin v1.0
  ```
  </TabItem>
</Tabs>

## 常见场景

### 1. 我想保存当前进度
<Card>
    ![保存进度示意图]
    1. 写完代码
    2. `git add .` (收集所有改动)
    3. `git commit -m "添加了xxx功能"` (保存进度)
    4. `git push` (上传到网上备份)
</Card>

### 2. 完了，我改错了，回滚之前的版本
<Card>
    1. `git log --oneline` (查看所有存档点)
    2. `git reset --hard 版本号` (回到指定的存档)
    
    就像游戏读取存档一样简单！
</Card>

### 3. 我想尝试一个新功能，但不确定是否可行
<Card>
    1. `git checkout -b test-new-feature` (创建新的测试分支)
    2. 在新分支上随便改
    3. 如果改砸了，直接删掉这个分支就好
    4. 如果改好了，可以合并回主分支
    5. 在命令 `git checkout -b test-new-feature` 中：
`git checkout -b` 是固定的命令格式（意思是"创建并切换到新分支"）
`test-new-feature` 是分支名称（这个名字你可以自由更改）
    
    就像游戏开了个新存档来尝试不同的剧情！
</Card>

## 注意事项

<CardGrid>
	<Card title="常备操作" icon="open-book">
		每天写代码前：`git pull` (下载最新代码)
        
        每天写完代码：`git push` (上传保存)
	</Card>

	<Card title="不确定就备份" icon="warning">
		要改重要的代码前，先创建新分支
        
        `git checkout -b backup-当前日期`
	</Card>
</CardGrid>
